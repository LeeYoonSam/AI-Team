# 기술 리서치

## 리서치 주제
$ARGUMENTS

## 리서치 프로세스

### 1. 범위 정의
- 리서치 목적 명확화
- 조사할 영역 정의
- 예상 산출물 명시

### 2. 내부 조사
**코드베이스 분석**:
- 기존 구현 패턴 탐색
- 관련 코드 검색
- 의존성 분석

**문서 검토**:
- CLAUDE.md 규칙 확인
- 기존 문서 검토
- ADR 확인

### 3. 외부 조사
**기술 문서**:
- 공식 문서 조회
- API 레퍼런스 확인

**모범 사례**:
- 업계 표준 조사
- 오픈소스 사례 분석

### 4. 분석 및 종합
- 발견사항 정리
- 장단점 분석
- 리스크 식별

### 5. 권장사항 도출
- 실행 가능한 권장사항
- 우선순위 지정
- 다음 단계 제안

## 산출물

**파일**: `state/outputs/research-{주제}.md`

```markdown
# 리서치 보고서: {주제}

## 요약
핵심 발견사항 요약

## 조사 범위
- 조사한 영역
- 사용한 소스

## 상세 분석

### 발견사항 1
...

### 발견사항 2
...

## 권장사항
1. [높음] ...
2. [중간] ...
3. [낮음] ...

## 리스크
- 잠재적 위험 요소

## 참조
- 참조 자료 목록

## 신뢰도
- 높음/중간/낮음
```

## 완료 후

1. 산출물을 `state/outputs/`에 저장
2. `state/project-state.json` 업데이트
3. 다음 단계 안내
