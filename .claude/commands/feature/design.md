# 시스템 설계

## 설계 대상
$ARGUMENTS

## 설계 프로세스

### 1. 요구사항 분석
- 기능 요구사항 정리
- 비기능 요구사항 식별
- 제약 조건 파악

### 2. 기존 아키텍처 분석
- 현재 시스템 구조 파악
- 관련 컴포넌트 식별
- 기존 패턴 확인

### 3. 설계 수립

**시스템 아키텍처**:
- 컴포넌트 구조
- 데이터 흐름
- 인터페이스 정의

**API 설계**:
- 엔드포인트 정의
- 요청/응답 스키마
- 에러 처리

**데이터 모델**:
- 엔티티 정의
- 관계 설정
- 스키마 설계

### 4. 트레이드오프 분석
- 대안 검토
- 장단점 분석
- 선택 근거

### 5. 문서화
- ADR 작성
- API 명세
- 다이어그램
