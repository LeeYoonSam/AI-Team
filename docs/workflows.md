# 워크플로우 및 작업 흐름

## 개요

이 문서는 멀티에이전트 시스템의 주요 워크플로우와 슬래시 커맨드 사용법을 설명합니다.

## 슬래시 커맨드 목록

### 프로젝트 관리

| 커맨드 | 설명 | 예시 |
|--------|------|------|
| `/project:init` | 프로젝트 초기화 | `/project:init web` |
| `/project:status` | 상태 확인 | `/project:status` |
| `/project:sync` | 문서 동기화 | `/project:sync` |

### 기능 개발

| 커맨드 | 설명 | 예시 |
|--------|------|------|
| `/feature:implement` | 기능 구현 오케스트레이션 | `/feature:implement 사용자 인증` |
| `/feature:research` | 기술 리서치 | `/feature:research JWT vs Session` |
| `/feature:design` | 시스템 설계 | `/feature:design 결제 시스템` |

### 리뷰

| 커맨드 | 설명 | 예시 |
|--------|------|------|
| `/review:code` | 코드 리뷰 | `/review:code staged` |
| `/review:security` | 보안 리뷰 | `/review:security all` |

### 테스트

| 커맨드 | 설명 | 예시 |
|--------|------|------|
| `/test:run` | 테스트 실행 | `/test:run unit` |
| `/test:coverage` | 커버리지 분석 | `/test:coverage` |

### Git

| 커맨드 | 설명 | 예시 |
|--------|------|------|
| `/git:commit` | 자동 커밋 | `/git:commit feat: 로그인 구현` |
| `/git:pr` | PR 생성 | `/git:pr 사용자 인증 기능` |

### 문서

| 커맨드 | 설명 | 예시 |
|--------|------|------|
| `/docs:sync` | 문서 동기화 | `/docs:sync` |
| `/docs:update` | 문서 업데이트 | `/docs:update architecture` |

## 기능 구현 워크플로우

### 전체 흐름

```
사용자 요청
    │
    ▼
┌──────────────────┐
│ /feature:implement │
│   (매니저 에이전트)  │
└────────┬─────────┘
         │
         ▼
    Phase 1: 기획
    ┌─────────────────────────────────────┐
    │                                     │
    ▼                ▼                   ▼
┌────────┐    ┌──────────┐    ┌─────────┐
│리서처   │    │아키텍트   │    │디자이너  │
│기술조사 │    │시스템설계 │    │UI/UX   │
└───┬────┘    └────┬─────┘    └────┬────┘
    │              │               │
    └──────────────┼───────────────┘
                   ▼
            계획 통합 및 검토
                   │
                   ▼
    Phase 2: 구현
    ┌─────────────────────────────────────┐
    │                                     │
    ▼                ▼                   ▼
┌────────────┐ ┌───────────┐ ┌──────────┐
│백엔드 개발자│ │프론트엔드  │ │모바일    │
│API 구현    │ │UI 구현    │ │앱 구현   │
└─────┬──────┘ └─────┬─────┘ └────┬─────┘
      │              │             │
      └──────────────┼─────────────┘
                     ▼
    Phase 3: 검증
    ┌─────────────────────────────────────┐
    │                                     │
    ▼                ▼                   ▼
┌────────────┐ ┌──────────┐ ┌───────────┐
│코드 리뷰어 │ │테스터     │ │보안 감사자│
│품질 검토   │ │테스트    │ │보안 점검  │
└─────┬──────┘ └────┬─────┘ └─────┬─────┘
      │             │             │
      └─────────────┼─────────────┘
                    ▼
    Phase 4: 배포
┌───────────────────────────────────────┐
│                DevOps                  │
│           CI/CD 및 배포                │
└───────────────────────────────────────┘
                    │
                    ▼
              완료 및 문서 업데이트
```

### 단계별 상세

#### Phase 1: 기획 (병렬 실행)

**리서처 에이전트**:
- 관련 기술 조사
- 기존 패턴 분석
- 권장사항 도출

**아키텍트 에이전트**:
- 시스템 설계
- API 계약 정의
- 데이터 모델 설계

**디자이너 에이전트** (UI 기능인 경우):
- UI/UX 설계
- 컴포넌트 명세
- 사용자 흐름 정의

#### Phase 2: 구현 (API 계약 기반 병렬)

**백엔드 개발자**:
- API 엔드포인트 구현
- 데이터베이스 작업
- 비즈니스 로직

**프론트엔드 개발자**:
- UI 컴포넌트 구현
- 상태 관리
- API 통합

**모바일 개발자** (필요 시):
- 모바일 앱 기능 구현

#### Phase 3: 검증 (병렬 실행)

**코드 리뷰어**:
- 코드 품질 검토
- 표준 준수 확인

**테스터**:
- 테스트 작성 및 실행
- 커버리지 분석

**보안 감사자**:
- 보안 취약점 분석
- OWASP 점검

#### Phase 4: 배포

**DevOps**:
- CI/CD 파이프라인 업데이트
- 배포 준비
